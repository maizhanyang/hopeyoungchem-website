### 1.3 添加错误边界和 404 页面

**问题分析**:
- 没有错误处理，组件崩溃白屏
- 缺少 404 页面，用户访问无效路径无反馈

**修改方案**:

```typescript
// src/components/ErrorBoundary.tsx
export class ErrorBoundary extends Component {
  state = { hasError: false }

  static getDerivedStateFromError() {
    return { hasError: true }
  }

  render() {
    if (this.state.hasError) {
      return <ErrorFallback />
    }
    return this.props.children
  }
}

// src/pages/NotFound.tsx
export function NotFound() {
  return (
    <div className="not-found">
      <h1>页面未找到</h1>
      <Link to="/">返回首页</Link>
    </div>
  )
}

// 在 App.tsx 中使用
<Routes>
  {/* 现有路由 */}
  <Route path="*" element={<NotFound />} />
</Routes>
```

**验证标准**:
- 故意抛出错误，显示友好错误页面
- 访问无效路径显示 404 页面

### 2.2 添加页面过渡动画

**问题分析**:

- 影响用户体验和专业感

**修改方案**:

```typescript
// src/components/PageTransition.tsx
import { motion, AnimatePresence } from 'framer-motion'

export function PageTransition({ children }: { children: React.ReactNode }) {
  return (
    <AnimatePresence mode="wait">
      <motion.div
        initial={{ opacity: 0, y: 20 }}
        animate={{ opacity: 1, y: 0 }}
        exit={{ opacity: 0, y: -20 }}
        transition={{ duration: 0.3 }}
      >
        {children}
      </motion.div>
    </AnimatePresence>
  )
}

// 在路由中使用
<Route path="/" element={
  <PageTransition><Home /></PageTransition>
} />
```

### 2.3 图片懒加载和优化

**问题分析**:
- 所有图片立即加载，浪费带宽
- 没有响应式图片，移动端加载大图

**修改方案**:

```typescript
// src/components/OptimizedImage.tsx
export function OptimizedImage({
  src,
  alt,
  width,
  height
}: ImageProps) {
  const [isLoaded, setIsLoaded] = useState(false)
  const imgRef = useRef<HTMLImageElement>(null)

  useEffect(() => {
    if (!imgRef.current) return

    const observer = new IntersectionObserver((entries) => {
      entries.forEach(entry => {
        if (entry.isIntersecting) {
          imgRef.current!.src = src
          observer.disconnect()
        }
      })
    })

    observer.observe(imgRef.current)
    return () => observer.disconnect()
  }, [src])

  return (
    <img
      ref={imgRef}
      alt={alt}
      width={width}
      height={height}
      loading="lazy"
      onLoad={() => setIsLoaded(true)}
      style={{ opacity: isLoaded ? 1 : 0.5 }}
    />
  )
}
```

**验证标准**:
- 图片进入视口才加载
- Lighthouse 性能分数提升

## 🔵 阶段三：SEO 与性能（2-3天）

### 3.1 动态 Meta 标签（关键）

**问题分析**:
- SPA 所有页面共享同一个 title/description
- 搜索引擎无法区分不同页面内容

**修改方案**:

```typescript
// src/hooks/useSEO.ts
export function useSEO(seoData: SEOData) {
  useEffect(() => {
    document.title = seoData.title

    const metaDescription = document.querySelector('meta[name="description"]')
    if (metaDescription) {
      metaDescription.setAttribute('content', seoData.description)
    }

    // Open Graph
    setMetaTag('og:title', seoData.title)
    setMetaTag('og:description', seoData.description)
    setMetaTag('og:type', 'website')
  }, [seoData])
}

// src/pages/Home.tsx
function Home() {
  const { t } = useTranslation()

  useSEO({
    title: '虹扬化工 - 高端材料粘接解决方案',
    description: '广州虹扬化工科技有限公司专注高端材料粘接...',
    keywords: '化工,粘接剂,材料科学'
  })

  return <HomeContent />
}
```

### 3.2 生成 Sitemap 和 Robots.txt

**问题分析**:
- 搜索引擎爬虫不知道有哪些页面
- 无法控制爬虫行为

**修改方案**:

```javascript
// scripts/generate-sitemap.js
import fs from 'fs'

const pages = [
  { url: '/', changefreq: 'weekly', priority: 1.0 },
  { url: '/about', changefreq: 'monthly', priority: 0.8 },
  { url: '/solutions', changefreq: 'weekly', priority: 0.9 },
  // ...
]

const sitemap = `<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
${pages.map(page => `
  <url>
    <loc>https://hongyang.com${page.url}</loc>
    <changefreq>${page.changefreq}</changefreq>
    <priority>${page.priority}</priority>
  </url>
`).join('')}
</urlset>`

fs.writeFileSync('public/sitemap.xml', sitemap)
```

```txt
# public/robots.txt
User-agent: *
Allow: /

Sitemap: https://hongyang.com/sitemap.xml
```

**在 package.json 添加脚本**:
```json
{
  "scripts": {
    "build": "npm run generate:sitemap && tsc -b && vite build",
    "generate:sitemap": "node scripts/generate-sitemap.js"
  }
}
```

**验证标准**:
- 访问 /sitemap.xml 显示完整站点地图
- Google Search Console 可提交成功

### 4.4 配置 CI/CD 流程

**问题分析**:
- 手动部署容易出错
- 没有自动化测试和质量检查

**修改方案**:

```yaml
# .github/workflows/ci.yml
name: CI/CD

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: 'npm'

      - run: npm ci
      - run: npm run lint
      - run: npm run test
      - run: npm run build

  deploy:
    needs: test
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npm run build

      - name: Deploy to Vercel
        uses: amondnet/vercel-action@v25
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
          vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
          vercel-args: '--prod'
```

**验证标准**:

export function trackError(error: Error, context?: string) {
  // 发送到错误追踪服务（如 Sentry）
  if (import.meta.env.PROD) {
    fetch('/api/errors', {
      method: 'POST',
      body: JSON.stringify({
        message: error.message,
        stack: error.stack,
        context,
        url: window.location.href,
        timestamp: new Date().toISOString()
      })
    })
  }
}

// 在 ErrorBoundary 中使用
componentDidCatch(error: Error) {
  trackError(error, 'ErrorBoundary')
  this.setState({ hasError: true })
}
